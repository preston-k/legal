Security Policy

Overview

We take user security seriously. To ensure that your data is stored securely, here are some of the security features that are in place when you visit any website in the PrestonKwei.com suite of apps. Any site with the domain, prestonkwei.com is subject to this privacy policy. These sites will be hereinafter be referred to as "Site", "Sites", or "Website". For any questions, comments, or concerns reguarding this security policy, please email us at infosec@prestonkwei.com.

HTTPS Encrypted Websites

All of our sites use HTTPS, a web standard that encrypts info when it gets sent to our site. When visiting our sites, you should NEVER get to a screen that says, "Your Connection is Not Secure". If you encounter this page, please take a screenshot and email it to infosec@prestonkwei.com, then close the site immediately.

Encrypted and Hashed User Information

We know that providing sensitve information to any site may be a cause of concern, so that's why all sensitive user information is encrypted and hashed using a 256bit hashing system. This ensures that even in the event of a data breach, it will be nearly impossible to return the original input. Sensitive user informaton includes, but is not limited to, emails, passwords, first name, last name, username, birthday, security question answers, billing information, and much more. Because we are unable to access the original data once it has been hashed, for forms that require for password, for example, hash the password you entered and compares it with the hashed password in our database.

Secure Data Storage

Database storage is handled by Google Cloud, who upholds strict data security measures to ensure your physical data is as secure as physically possible. This involves a variety of physical and digital security methods. Profile photos and email processing is handled by Amazon Web Services. Amazon Web Services is a third party, who provides the best cloud security out of all the platforms. They run the servers that handle email for our platforms. As with any third party, you can learn more about their data and security policies by visting their respective websites.

Restricted Database Access

Database access is highly restricted. Only the developers who MUST have access to the user data are allowed to access it. Even so, developers are restricted to ONLY the data that they MUST have access to in order to perform their duties.
